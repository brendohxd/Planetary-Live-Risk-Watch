# Fork this for your country

**If you care about the place you live — take this repo and build the local version.**

One serious person per country is enough to start.

The more *local*, *specific*, and *plain-spoken* the information becomes, the more people can actually understand it and act. Global jargon loses the majority. Local names, local agencies, local evacuation habits, and local memory of the last bad season win trust.

### What we need from you

1. **Fork** this repository
2. Rebuild it for **your** country or region
3. Put **your** official warning services first (weather, fire, flood, quake, civil defence)
4. Write in the everyday language people there already use
5. Add what only locals know: evacuation culture, trusted numbers, state/provincial quirks, history people recognise

The deeper and more specific you go, the better real safety advice reaches ordinary people — not just specialists.

Help improve this Australia build too if that is your home. Open issues. Send pull requests. Challenge weak language. Demand clarity when something is serious.

## This build is Australia-first

[Earth Risk Watch · Australia](https://brendohxd.github.io/Planetary-Live-Risk-Watch/) is a plain-language public awareness page. It prioritises Australian context, BOM, and state emergency practice.

It is **not** an official warning system.

### Official warnings always win

Use these first:

- [Bureau of Meteorology](https://www.bom.gov.au/)
- Your **state emergency service** / fire service apps
- Local council and police directions in an emergency

### Why Australia first?

People prepare better when they recognise **their** weather service, **their** fire danger ratings, and **their** last bad summer. Abstract global dashboards often fail that test.

### Language rule

Prefer plain English. If a risk is **serious**, say **serious**. Do not hide severity behind technical terms. The majority will not act on language they cannot parse.
